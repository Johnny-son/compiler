# MiniC 编译器可执行开发清单

本文档按当前代码结构维护。每个阶段都尽量保持一个原则：**语法、AST、IR、后端、测试同步推进**。

当前后端目录是 `src/backend`，不是旧版文档中提到的 `src/backend/riscv64`。

## 执行原则

- 先扩大语言能力，再做优化和寄存器分配。
- 每次只解锁一小批测试，便于定位回归。
- 每次改动后都跑：

```sh
bash scripts/run_ci_tests.sh --all --compiler ./build/compiler
```

- 新增错误诊断时，同步更新：
  - `docs/Architecture/diagnostics/错误码表.md`
  - `scripts/ci_test_plan.txt`
  - 对应 `test/.../*.c` 中的 `@expected-error`

## 阶段 2：`const` 与常量定义（建议下一步）

### 为什么优先做

当前不少被注释测试的第一失败点是 `const`，例如：

```c
const int a = 10;
```

先支持标量 `const int`，可以解锁一批基础算术测试，并且改动范围比数组、控制流更小。

### 目标

- 支持全局和局部 `const int` 标量定义。
- 支持常量在表达式中使用。
- 禁止给常量重新赋值。
- 保持全局初始化只接受常量整数表达式。

### 核心文件

- `src/frontend/antlr4/MiniC.g4`
- `src/frontend/CSTVisitor.cpp`
- `src/frontend/include/AST.h`
- `src/ir/IRGenerator.cpp`
- `src/ir/Module.cpp`
- `src/ir/include/Value.h`
- `src/ir/Values/ConstInt.h`

### 主要任务

- 扩语法：`constDecl`、`constDef`，或在现有 `varDecl` 上增加 `const` 修饰。
- AST 中记录声明是否为常量。
- IR/符号层区分“可写变量”和“只读常量”。
- 赋值语句检查左侧是否为常量。
- 为常量重复定义、常量赋值等场景补诊断测试。

### 建议先开的测试

- `2023_func_12_addc.c`
- `2023_func_13_sub2.c`
- `2023_func_16_mulc.c`
- `2023_func_18_divc.c`
- `2023_func_06_const_var_defn2.c`
- `2023_func_07_const_var_defn3.c`

### 完成标志

- 标量 `const int` 能生成正确 IR 和 ASM。
- 常量被赋值时能报清晰错误。
- `scripts/ci_test_plan.txt` 解锁对应用例后 `--all` 全绿。
