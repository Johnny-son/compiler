# MiniC 编译器可执行开发清单

本文档将当前 `minic` 编译器的完善工作拆分为可执行阶段。每个阶段都给出：

- 目标能力
- 建议修改的核心文件
- 建议优先打开的测试用例
- 完成标志

建议执行原则：

- 先补语言能力，再补优化与寄存器分配
- 每完成一类能力，就在 `scripts/ci_test_plan.txt` 中逐步解锁对应测试
- 优先保证 `frontend -> ir -> backend -> test` 四段同步推进，避免某一段先走太远

## 推荐执行节奏

建议按下面的节奏推进，而不是并行大面积铺开：

1. 阶段 1 做完后，只打开函数参数相关测试
2. 阶段 2 做完后，打开 `if` 和短路相关测试
3. 阶段 3 做完后，打开 `while/break/continue`
4. 阶段 4 和阶段 5 做完后，再系统打开数组与常量相关测试
5. 阶段 6 和阶段 7 用来把复杂样例跑稳
6. 阶段 8 稳定后，再进入阶段 9
