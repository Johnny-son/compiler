## 错误码分段规则

```text
E + 分类编号 + 顺序号
```

| 分段 | 含义 |
|------|------|
| E1xxx | IR 生成层错误 |
| E10xx | 节点/符号基础错误 |
| E11xx | 函数定义和形参相关错误 |
| E12xx | 函数调用相关错误 |
| E13xx | 全局变量与初始化相关错误 |
| E14xx | 变量定义相关错误 |
| E15xx | 调用约定与 IR 一致性错误 |

## 输出格式

IR 层错误统一使用：

```text
IR错误[E####] 第N行 分类: 详细原因
IR错误[E####] 未知行 分类: 详细原因
```

行号未知时输出 `未知行`，不要输出负数行号。

## 常见错误详情

### E1000 — 不支持的 AST 节点类型

触发条件：`IRGenerator` 遇到没有注册翻译函数的 AST 节点。

示例：

```text
IR错误[E1000] 第3行 节点分发: 不支持的AST节点类型(42)
```

### E1010 — 标识符未定义

触发条件：表达式中引用了当前作用域链找不到的变量。

对应测试：`test/stage1_params/param_undef_var.c`

### E1102 — 函数重复定义

触发条件：同名函数已经存在于 `Module` 函数表。

对应测试：`test/stage1_params/param_func_redef.c`

### E1112 — 形参映射失败

触发条件：形参注册为局部变量失败，常见原因是同一个函数的形参重名。

对应测试：`test/stage1_params/param_duplicate.c`

### E1200 — 函数未定义或未声明

触发条件：调用点找不到目标函数。

对应测试：`test/stage1_params/param_call_undef.c`

### E1201 — 调用参数个数不匹配

触发条件：实参数量与函数形参数量不同。

对应测试：

- `test/stage1_params/param_arg_too_many.c`
- `test/stage1_params/param_arg_too_few.c`

### E1202 — 调用参数类型不匹配

触发条件：实参类型与对应形参类型不同。

当前语言子集类型较少，因此该错误主要为后续类型系统扩展预留。

### E1301 — 全局初始化非常量

触发条件：全局变量初始化表达式不是当前支持的常量整数表达式。

对应测试：`test/stage1_params/param_global_init_nonconst.c`

### E1303 — 常量赋值

触发条件：尝试对 `const` 常量重新赋值。

对应测试：需要在 `test/contesttestcases` 中新增

### E1304 — 常量未初始化

触发条件：`const` 常量定义时没有提供初始值。

对应测试：需要在 `test/contesttestcases` 中新增

### E1400 — 变量重复定义

触发条件：同一作用域中重复声明同名变量。

对应测试：`test/stage1_params/param_redeclare_local.c`

## 维护说明

1. 新增错误码时，优先使用 `report_ir_error()` 或 `report_module_error()` 保持格式一致。
2. 同步更新本表和诊断测试。
3. 测试匹配重点是 E 码、行号和关键符号；错误说明文字可以适度调整。
4. 不建议在表格里维护精确源码行号，行号容易随普通编辑漂移。
