# 运行方式

## 1. 构建前准备

建议环境（与 CI 基本一致）：

- CMake 3.20+
- C++17 编译器（`g++` 或 `clang++`）
- Java Runtime（用于 ANTLR 代码生成）
- Graphviz 开发库（用于 AST 图片）
- ANTLR4 C++ Runtime（需与本机环境兼容；ANTLR 工具 jar 已随仓库放在 `thirdparty/antlr-4.13.2-complete.jar`）

## 2. 项目构建

在仓库根目录执行：

```sh
cmake -S . -B build
cmake --build build -j
```

构建成功后，可执行文件默认位于：

```sh
./build/compiler
```

## 3. 常用运行命令

以下示例使用测试用例 `2023_func_00_main.c`。

### 3.1 生成 AST 图片

```sh
./build/compiler -S -T -A -o ./output.png \
  ./test/contesttestcases/2023_function/2023_func_00_main.c
```

### 3.2 生成 IR（LLVM 文本）

```sh
./build/compiler -S -I -A -o ./output.ll \
  ./test/contesttestcases/2023_function/2023_func_00_main.c
```

### 3.3 生成 RISCV64 汇编

```sh
./build/compiler -S -A -o ./output.s \
  ./test/contesttestcases/2023_function/2023_func_00_main.c
```

## 4. 常用参数说明

- `-S`：开启编译输出流程（必选）
- `-T`：输出 AST
- `-I` / `-L`：输出 IR
- `-t RISCV64`：指定目标架构（默认 RISCV64）
- `-o <path>`：指定输出文件
- `-c`：汇编中附带 IR 注释

未指定 `-T/-I/-L` 时，默认输出汇编。

## 5. 快速自检

```sh
bash scripts/run_ci_tests.sh --all --compiler ./build/compiler
```

如果 `ast`、`ir`、`asm`、`err` 均显示 `all passed`，说明当前环境和构建产物可正常使用。
