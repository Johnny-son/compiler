# 架构设计

本文档描述当前源码实现，而不是完整 MiniC 目标形态。更细的逐文件说明见 [Architecture 文档导航](./Architecture/README.md)。

## 1. 总体流水线

项目采用三段式编译流程：

```text
frontend -> ir -> backend
```

真实数据流如下：

```text
source(.c)
  -> ASTGenerator
  -> ANTLR4 Lexer/Parser
  -> CSTVisitor
  -> ast_node 树
  -> IRGenerator
  -> Module / Function / Instruction / Value
  -> BackendDriver
  -> IRAdapter
  -> FrameLayoutBuilder
  -> InstructionSelector
  -> AsmPrinter
  -> RISC-V64 assembly(.s)
```

主控逻辑在 `src/main.cpp` 的 `compile()`：

1. `ArgsAnalysis` 解析命令行参数。
2. `ASTGenerator` 生成 AST。
3. 如果指定 `-T`，通过 `OutputAST` 输出 AST 图片后结束。
4. `Module` 保存函数、变量、常量、作用域等模块级上下文。
5. `IRGenerator` 把 AST 翻译成线性 IR。
6. 如果指定 `-I`/`-L`，`Module::outputIR` 输出 LLVM 风格文本 IR 后结束。
7. 默认进入 `BackendDriver`，输出 RISC-V64 汇编。

## 2. 当前语言能力边界

当前支持的是基础整数子集，适合作为后续迭代基线：

- `int` 标量变量：全局变量、局部变量
- `const int` 标量常量：全局常量、局部常量
- 函数：定义、标量形参、调用、返回值
- 语句：声明、赋值、块语句、表达式语句、`return`
- 表达式：整数常量、变量引用、函数调用、`+ - * / %`
- 内置函数：`putint(int)`、`getint()`
- 语义诊断：重复定义、未定义变量/函数、参数数量和类型不匹配、全局初始化限制、常量赋值检查

暂未完整支持：

- `const` 数组
- 数组与下标访问
- `if` / `while` / `break` / `continue`
- 关系表达式、逻辑表达式、短路求值
- `float`
- 优化管线和独立寄存器分配

## 3. Frontend

前端负责把源码文本变成自定义 AST。

关键文件：

- `src/frontend/antlr4/MiniC.g4`：当前 MiniC 子集的词法和语法定义。
- `src/frontend/ASTGenerator.cpp`：ANTLR4 入口，负责读取文件、构造 lexer/parser、触发 CST visitor。
- `src/frontend/CSTVisitor.cpp`：把 ANTLR CST 转成项目自定义 AST。
- `src/frontend/include/AST.h`：AST 节点类型、节点字段和工厂函数。
- `src/frontend/Graph.cpp`：用 Graphviz 输出 AST 图片。

当前 AST 节点不仅保存语法结构，也带有中端翻译会复用的字段：

- `blockInsts`：该节点翻译出的 IR 指令片段。
- `val`：该节点表达式求值后的 `Value`。
- `needScope`：控制 block 翻译时是否自动进出作用域。

## 4. IR

IR 层是当前项目的中枢，负责两件事：

1. 把 AST 翻译成线性指令序列。
2. 管理符号、作用域、类型和值之间的依赖关系。

关键文件：

- `src/ir/IRGenerator.cpp`：AST 到 IR 的核心翻译器。
- `src/ir/include/Module.h`、`src/ir/Module.cpp`：模块级容器，管理函数、全局变量、常量池和作用域。
- `src/ir/include/Function.h`：函数对象，持有形参、局部变量和 `InterCode`。
- `src/ir/include/Instruction.h`：IR 指令基类与操作码。
- `src/ir/include/Value.h`、`User.h`、`Use.h`：值系统与 def-use 关系。
- `src/symboltable/ScopeStack.*`：变量作用域栈。

当前接通的 IR 指令包括：

- `entry` / `exit`
- `label` / `goto`
- `assign`
- `add` / `sub` / `mul` / `div` / `mod`
- `call`
- `arg` 类型仍保留，但当前函数调用主要通过 `FuncCallInstruction` 操作数承载实参

## 5. Backend

后端目标是输出 RISC-V64 汇编。它不再使用旧的 `backend/riscv64` 目录结构，当前实现集中在 `src/backend`。

关键文件：

- `src/backend/include/BackendDriver.h`、`src/backend/BackendDriver.cpp`：后端总入口。
- `src/backend/include/IRAdapter.h`、`src/backend/IRAdapter.cpp`：为后端提供 `Module/Function/Instruction/Value` 的只读视图。
- `src/backend/include/FrameLayout.h`、`src/backend/FrameLayout.cpp`：计算函数栈帧布局。
- `src/backend/include/InstructionSelector.h`、`src/backend/InstructionSelector.cpp`：把 IR 指令翻译成 RV64 指令模板。
- `src/backend/include/Asm.h`、`src/backend/Asm.cpp`：结构化汇编对象与最终文本打印。

后端当前链路：

```text
Module
  -> IRAdapter::adapt
  -> IRModuleView
  -> BackendDriver 遍历全局变量和函数
  -> FrameLayoutBuilder::build
  -> InstructionSelector::run
  -> AsmPrinter::printFunction
```

当前策略比较朴素：

- 局部变量、形参和 IR 临时结果统一分配栈槽。
- 栈槽最小 8 字节，函数栈帧按 16 字节对齐。
- 参数寄存器使用 `a0-a7`，返回值使用 `a0`。
- 指令选择固定使用少量工作寄存器，如 `t0/t1/t2/t3`。
- 暂无复杂寄存器分配器。

## 6. 测试与迭代节奏

当前基线测试由 `scripts/ci_test_plan.txt` 控制：

```sh
bash scripts/run_ci_tests.sh --all --compiler ./build/compiler
```

建议后续按“一个语言能力 + 一小批测试”的节奏迭代：

1. 修改 `MiniC.g4` 扩语法。
2. 修改 `CSTVisitor.cpp` 和 `AST.h` 扩 AST。
3. 修改 `IRGenerator.cpp` 和必要的 IR 指令/Value/Type。
4. 修改 `InstructionSelector.cpp` 和必要的后端布局逻辑。
5. 在 `scripts/ci_test_plan.txt` 解锁对应测试。
6. 跑 `--all` 确认旧能力没有回归。
