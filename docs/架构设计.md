# 架构设计

## 1. 总体流水线

项目采用三段式编译流程：

```text
frontend -> ir -> backend
```

数据流如下：

```text
source(.c)
  -> ANTLR 词法/语法分析
  -> CSTVisitor 生成 AST
  -> IRGenerator 生成线性 IR
  -> CodeGeneratorRiscv64 生成汇编(.s)
```

## 2. 主流程入口

主控逻辑在：`src/main.cpp`

核心阶段：

1. 参数解析（`ArgsAnalysis`）
2. 前端生成 AST（`ASTGenerator`）
3. 中端生成 IR（`IRGenerator` + `Module`）
4. 后端生成 RISCV64 汇编（`CodeGeneratorRiscv64`）

## 3. Frontend

关键文件：

- `src/frontend/antlr4/MiniC.g4`：语法定义
- `src/frontend/ASTGenerator.cpp`：ANTLR 入口，负责得到 CST
- `src/frontend/CSTVisitor.cpp`：将 CST 转为 AST
- `src/frontend/include/AST.h`：AST 节点结构定义

## 4. IR（中间表示）

关键文件：

- `src/ir/IRGenerator.cpp`：AST 到 IR 的核心翻译
- `src/ir/include/Module.h`：模块级上下文（函数、全局变量、作用域）
- `src/symboltable/ScopeStack.*`：作用域栈
- `src/ir/include/Instruction.h`：IR 指令基类

实现方式：

- 通过 AST 节点类型到处理函数的映射表分发翻译逻辑。
- 每个 AST 节点会产出对应的 `Value` 与 `InterCode` 片段。

## 5. Backend（RISCV64）

关键文件：

- `src/backend/riscv64/CodeGeneratorRiscv64.cpp`：后端主入口
- `src/backend/riscv64/InstSelectorRiscv64.cpp`：IR 到目标指令选择
- `src/backend/riscv64/ILocRiscv64.cpp`：汇编指令发射
- `src/backend/riscv64/SimpleRegisterAllocator.cpp`：简单寄存器分配

当前寄存器分配策略较基础，后续可替换为线性扫描或图着色等方案。

## 6. 当前状态与扩展方向

当前框架已支持从 MiniC 子集到 IR/ASM 的完整链路，并具备 CI 测试入口。

可优先扩展方向：

1. 语义检查完善（定义/使用顺序、类型一致性等）
2. IR 优化 pass（常量折叠、死代码删除等）
3. 更完善的寄存器分配与调用约定处理
