# 测试说明

## 1. 测试数据位置

默认测试根目录：

```sh
test/contesttestcases
```

常见文件：

- `*.c`：输入源码
- `*.in`：标准输入（可选）
- `*.out`：期望输出
- `lib/std.c`：运行时库（链接用）

## 2. 测试脚本入口

统一使用：

```sh
scripts/run_ci_tests.sh
```

支持三种模式：

- `-ast`：检查 AST 文件是否生成
- `-ir`：编译 IR、链接本地可执行并比对输出
- `-asm`：编译 RISCV64 汇编、交叉链接并在 QEMU 下运行比对

## 3. 常用命令

### 3.1 指定单个或多个用例

```sh
bash scripts/run_ci_tests.sh -ir 2023_func_00_main.c --compiler ./build/compiler
bash scripts/run_ci_tests.sh -asm 2023_func_01_var_defn2.c 2023_func_02_var_defn3.c --compiler ./build/compiler
```

### 3.2 按目录跑

```sh
bash scripts/run_ci_tests.sh -ir 2023_function --compiler ./build/compiler
```

### 3.3 跑测试计划（CI 同款）

```sh
bash scripts/run_ci_tests.sh --all --compiler ./build/compiler
```

`--all` 会读取：

```sh
scripts/ci_test_plan.txt
```

## 4. 如何新增测试

1. 在 `test/contesttestcases/...` 下添加 `xxx.c`。
2. 按需要补 `xxx.in`（有输入时）。
3. 补 `xxx.out`（期望输出）。
4. 在 `scripts/ci_test_plan.txt` 的 `ir:`/`asm:`/`ast:` 对应分区添加该用例。
5. 本地执行 `run_ci_tests.sh` 验证。

## 5. 输出比对规则

脚本会将“程序标准输出”与“进程返回码”一起写入结果文件，再与 `.out` 比较。

因此如果用例没有 `printf` 输出，你常看到的 `.out` 只有一个数字（通常是 `main` 的返回值）。

## 6. 常见环境依赖

- `-ir` 需要本地 C 编译器（`clang` 或 `cc`）
- `-asm` 需要：
  - `riscv64-linux-gnu-gcc`
  - `qemu-riscv64-static` 或 `qemu-riscv64`
